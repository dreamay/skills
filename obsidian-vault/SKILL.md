# Obsidian 知识库管理（通用版）

适用于任意电脑、任意路径的 Obsidian 知识库。所有规则自包含在本 skill 中，无需依赖 Vault 内的特定文件。

---

## 第一步：定位 Vault

每次操作前，先确定 Vault 路径：

1. **用户指定了路径** → 直接使用
2. **用户未指定** → 询问用户 Vault 在哪里
3. **已初始化过的 Vault** → 检查是否存在 `CONTEXT.md`，读取其中的根目录路径确认

确定路径后，检查 `CONTEXT.md` 是否存在：
- **存在** → 已初始化的 Vault，读取 CONTEXT.md 获取结构信息，然后按需操作
- **不存在** → 全新 Vault，执行下方「首次初始化」

---

## 首次初始化

当目标 Vault 尚未初始化（无 `CONTEXT.md`）时，自动创建完整结构。

### 初始化内容

1. **创建目录树**（见 [templates.md](templates.md) 「目录树」章节）
2. **创建配置文件**：
   - `.user-preferences.md` — 用户偏好（见 templates.md）
   - `.ai-instructions.md` — AI 行为规范（见 templates.md）
   - `CONTEXT.md` — 全局索引入口（见 templates.md）
3. **创建模板文件**：`_Templates/` 下 14 个模板（见 templates.md）
4. **创建指令文件**：`_Prompts/` 下 7 个指令文件（见 templates.md）
5. **创建初始 MOC**：`07-MOC/Vault总览.md` 和 `07-MOC/最近更新.md`

初始化完成后告知用户：Vault 结构已就绪，可以开始使用了。

## 核心原则（必须遵守）

1. **只写一篇**：用户问什么就写什么，不衍生创建关联文件
2. **外链引用**：相关概念用外部 URL，不创建内部空文件。只有目标文件已存在时才用 `[[wiki link]]`
3. **精简总结**：知识卡片 800-1000 字，图文并茂，不写论文
4. **图文并茂**：架构图用 Mermaid，对比用表格，流程用 Mermaid 时序图
5. **面向快速理解**：假设用户是技术人员，直接进入重点

---

## 写作风格

- 中文为主，技术术语保留英文（如 Namespace、Cgroups）
- 禁止套话："值得注意的是"、"本文将介绍..."、"总之"、"综上所述"
- 不写教科书式开头和结尾
- 代码示例只放最核心的片段，不要贴完整配置

## Frontmatter 规范

每篇笔记必须包含：

```yaml
---
type: 笔记类型    # 见下方枚举
created: YYYY-MM-DD
updated: YYYY-MM-DD
tags: []
ai_generated: true
---
```

### type 枚举与目标文件夹

| type | 目标文件夹 | 命名格式 |
|------|-----------|---------|
| tech-card | 03-学习/技术专题/{子领域}/ | 概念名称.md |
| book-note | 03-学习/读书笔记/ | 书名-读书笔记.md |
| tech-deep | 03-学习/技术专题/{子领域}/ | 主题名称.md |

| paper-note | 03-学习/论文解读/ | 论文标题-解读.md |

| meeting | 02-工作/会议记录/ | YYYY-MM-DD-主题.md |

## 知识卡片结构（最常用）

```markdown
## 一句话定义
（1-2 句话说清楚）

## 核心原理
（Mermaid 图或表格 + 简短说明）

## 关键命令/代码
（最常用的 5-10 行，不贴大段配置）

## 优缺点速查
| 优点 | 缺点 |
|------|------|
| ... | ... |

## 适用场景
（3-5 个，每个一句话）

## 延伸阅读
- [标题](外部URL)
```

**字数**：800-1000 字，**禁止**超过 1200 字。

---

## 链接策略

**默认用外部 URL**：
```markdown
## 延伸阅读
- [Kubernetes 官方文档](https://kubernetes.io/docs/)
```

**仅在以下情况用 `[[wiki link]]`**：
1. 目标文件已存在于知识库中
2. 用户明确要求
3. 概念高度相关且知识库中已有

## 操作流程

### 写入新笔记后

1. 确认 frontmatter 完整
2. 延伸阅读使用外部 URL
3. 更新 `07-MOC/` 对应索引
4. 更新 `07-MOC/最近更新.md` 摘要区
5. 添加合适的 tags（从已有标签选择）

### 文件已存在时

**禁止直接覆盖**。先询问用户：
- **更新补充**：在现有基础上补充
- **完全重写**：覆盖旧内容
- **取消**

### 删除文件时（5 步完整执行）

1. 删除目标文件
2. 扫描 07-MOC/ 移除断链
3. 扫描其他笔记清理引用
4. 修正 CONTEXT.md 和 Vault总览 的统计
5. 在"最近更新"记录删除日志

### 知识库同步

触发词："帮我同步一下知识库"

1. 扫描实际文件统计数量
2. 修正统计数字
3. 新文件添加到 MOC（**只增不删用户文件**）
4. 清理确实不存在的断链
5. 检查笔记内断链并提醒
6. 在"最近更新"记录同步结果

### 定期校验

- **每天第一次操作**：自动完整校验（统计 + MOC）
- **其他时间**：仅用户说"同步"时才校验
